    • ***Web & HTML for Beginners

    Definitions
    Internet is the global system of interconnected computer networks.
    The web is the system we use to access the Internet.
    HTML stands for Hyper Text Markup Language which is the main document type of the web, invented in 1990s.
    Web browser is a program (Internet Explorer , FireFox, Google Chrome, etc) that is located on the computer that is used to display files on the web.
    Server is a computer designed to process requests and deliver data to other computers over a local network or the Internet.
    HTTP stands for Hyper Text Transfer Protocol which is the set of rules for transferring files (text, images, sound, video, and other multimedia files) on the World Wide Web.
    How the Web Works?
    When a user opens a web browser and writes www.udacity.com, the computer sends an HTTP request to a server. The server finds the HTML document and sends it back to the user's computer , the web browser interprets the page and displays it on the user's screen. Click this video to watch the detailed explanation.
    HTML
    HTML has "Text content" (what you see), "markup" (what it looks), "references to other documents" (images, videos, etc), "links to other pages".
    HTML consists of a series of codes typed into a text-file by the programmer;
    • TAGs - use < angle-brackets >,most tags have opening and closing tag, but some called "void" tag which does not require a closing tag.
    • Element - includes opening tag & content & closing tag. Elements can be inline or block; Block elements create an "invisible box" around the content inside of them, such as "p", "div", "form". "a", "span", "br", "img" and "strong" are all inline elements.
    • Attribute - provide additional information about an element , are specified in the opening tag.
    The Structure of HTML document : 
             <html>
              <head> 
                 <title> TITLE </title>
              </head>
              <body>  
                 <name> CONTENT </name>
              </body> 
          </html>

    ***HTML & CSS & DOM

    WebPage ingredients
    The browser turns HTML "tags" into "elements" that form a "tree". It knows how to do that because of the DOM "Document Object Model", which is a standard convention for representing and interacting with elements in HTML. Each HTML tag creates an element in the DOM that the browser uses to display the page.
    The document written in HTML defines the structure and content of the page.To define the style, you have to use another language; CSS "Cascading Style Sheets". CSS is used for describing the look and formatting of a document written in a markup language.
    DOM -a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.
    Text Editors for coding
    • ScratchPad : beginner friendly, least powerful, sharing is not possible, HTML & CSS
    • Codepen : powerful, easy for sharing but account needed, HTML & CSS & JavaScript
    • Sublime : most powerful,but not quite friendly, difficult to share, all languages
    Avoiding Repetition
    Coding principle is don't repeat yourself (DRY). Avoiding Repetition makes it easier to create & maintain & expand, less likely for errors and mistakes. Also, it makes it faster loading of web pages, enables the clients to save money on bandwidth and development costs.

    ***Python for Beginners

    Introduction to Programming
    A computer is a machine that can execute a program. With the right program, a computer can do any mechanical computation you can imagine.
    A program describes a very precise sequence of steps. Since the computer is just a machine, the program must give the steps in a way that can be executed mechanically. That is, the program can be followed without any thought.
    A programming language is a language designed for producing computer programs. A good programming language makes it easy for humans to read and write programs that can be executed by a computer. Programmin languages are needed because natural languages are ambiguous and verbose.
    Python is a programming language. The programs that we write in the Python language will be the input to the Python interpreter, which is a program that runs on the computer. The Python interpreter reads our programs and executes them by following the rules of the Python language.
    Grammatical structure is very strict, has to match the rules exactly, otherwise interpreter wouldn’t understand it, and gives error.
    Python expressions: 
              Expression    → Expression Operator Expression
          Expression    → Number
          Operator      + - * /
          Number        0, 1, ...

    Python problem solving starts with "not panicking". Understanding inputs, outputs & relationships, and creating examples are the basics of it. Starting to write as “pseudocode”, not exactly detailed correct code, helps to start solving.


    The trick to dealing with programming mistakes (called "bugs") is not to avoid making them entirely. That's impossible. The trick is to develop a system for finding and fixing them. These strategies help for debugging:
    1. Examine error messages when programs crash (last line)
    2. Work from example code
    3. Make sure examples work
    4. Check (print) intermediate results
    5. Keep and compare old versions
    Variables & Strings & Lists
    Variables give programmers a way to give names to values. Variables can be useful to programmers in many ways:
    • They improve code readability by using names that make sense to humans
    • They give us a way to store the value of important data
    • They give us a way to change the value
    The way to introduce a variable is using an assignment statement. We have a name,followed by an equal symbol, followed by an expression:
    Name = Expression
    A string is just a sequence of characters surrounded by quotes. ‘ or “ can be used, but they have to be same at the beginning and end. Examples : 
              print 4                 >>> output : 4
          print "4"               >>> output : 4
          print 4 + 4             >>> output : 8
          print "4" + "4"         >>> output : 44   note: 4 is a number, “4” is a string        
          print "hello’           >>> gives error because of the quotes types
    One of the things we can do with strings that we can't do with numbers, is we can extract subsequences from the strings.If we have a string, we can use the square brackets to extract parts of that string.Indexing a string starts from 0-1-2-3 etc., we can also use negative indexing. Examples: 
              ilkin[2]                >>> output : k
          ilkin[-2]               >>> output : i
          ilkin[2:3]              >>> output : k
          ilkin[2:4]              >>> output : ki
          ilkin[2:]               >>> output : kin
    In addition to using single quotes (') or double quotes (") to create a string, you can also use TRIPLE quotes to create a multi-line string.
    A List is a more powerful and general type of structured data. Compared to a string where all of the elements must be characters, in a list the elements can be anything you want such as characters, strings, numbers or even other lists. 
              String                  List
          s = 'yabba!'            p = ['y','a','b','b','a','!']                  
          s[2:4] -> 'bb'          p[2:4] -> ['b','b']
    Input→Function→Output
    Functions (procedure) take inputs, and produce outputs. Function blocks begin with the keyword def followed by the function name and parentheses ( ( ) ). Any input parameters (arguments or operands) should be placed within these parentheses. You can also define parameters inside these parentheses. Every function starts with a colon (:) and is indented.
    Functions help programmers avoid repetition by re-using the same definition.
    The statement return [expression] exits a function, optionally passing back an expression to the caller. A return statement with no arguments is the same as return None.
    Examples: 
              def square(n):
            return n * n
          print square (5)       >>> output : 25
          def abbaize(a,b):
            return a + b + b + a
          print abbaize('a','b') >>> output : abba
          def abbaize(a,b):
            c = a + b + b + a
          print abbaize('a','b') >>> output : None
    Control Flow & Loops & Operators
    Python provides lots of different operators for comparisons, such as greater than (>), less than (<), equal to (==) and not equal to (!=). The output of a comparison is not a number, it is a Boolean value (True or False).
    ‘==’ for equality comparison, ‘=’ for assignment 
              i = 21  assingns  21 value to i 
          i==21   returns True or False
    If statement helps to make decision; "if" followed by a comparison (test expression), followed by a colon. And then, inside the "if", there is the block, the block is the code that will run when the test expression is True.
    If the test expression doesn't evaluate to True, the block doesn't execute. Example : 
              def bigger(a,b):
            if a>b:
              r=a
            else:
              r=b
            return r
          print bigger(2,9)      >>> output:9
    OR operator displays True if either of the expression True. Only False OR False give False.
    While loop is similar to If statement, "While" is followed by a test expression, followed by a colon, and then a "block" has a sequence of instructions. If executes only 1 time, While executes until test expression is False. Example : 
              def print_numbers (n):
            i = 1
            while i < n:
              print i
              i = i + 1
            return i
    The append method adds a new element to the end of a list. The append method mutates the list that it is invoked on, it does not create a new list. The syntax for the append method is: 
              <list>.append(<element>)
    The + (concatenation) operator can be used with lists and is very similar to how it is used to concatenate strings. It produces a new list, it does not mutate either of the input lists. 
              <list> + <list> → <list>
    The len (length) operator can be used to find out the length of an object. The len operator works for many things other than lists, it works for any object that is a collection of things including strings. The output from len is the number of elements in its input. 
              len(<list>) → <number>
    Example: 
              p = [1,2]
          q = [3,4]
          p.append(q)
          p = p + [4,5]
          len(p)                 >>> output: 5
    In Python, there is a more convenient way to loop through the elements of a list: for loop. The loop goes through each element of the list in turn, assigning that element to the <name> and evaluating the <block>. Using the for loop helps to use less code than while loop. 
              for <name> in <list>:
          <block>
    Example: 
              def find_element(p,t):
              i=0
              for e in p:
                if e==t:
                  return i
                i=i+1
                return -1

    ***Python - Object Oriented Programming (OOP)

    Introduction to OOP
    Object Oriented Programming (OOP) is a programming language model, organized around OBJECTs rather than "actions" and DATA rather than "logic". Historically, a program has been viewed as a logical procedure that takes input data, processes it, and produces output data.
    The first step in OOP is to identify all the objects the programmer wants to manipulate and how they relate to each other, an exercise often known as data modeling. Once an object has been identified, it is generalized as a class of objects which defines the kind of data it contains and any logic sequences that can manipulate it. Each distinct logic sequence is known as a method. Objects communicate with well-defined interfaces called messages.

    The concepts and rules used in Object Oriented Programming provide these important benefits:

    • The concept of a data class makes it possible to define sub-classes of data objects that share some or all of the main class characteristics. Called inheritance, this property of OOP forces a more thorough data analysis, reduces development time, and ensures more accurate coding.
    • Since a class defines only the data it needs to be concerned with, when an instance of that class (an object) is run, the code will not be able to accidentally access other program data. This characteristic of data hiding provides greater system security and avoids unintended data corruption.
    • The definition of a class is reuseable not only by the program for which it is initially created but also by other Object Oriented Programs (and, for this reason, can be more easily distributed for use in networks).
    • The concept of data classes allows a programmer to create any new data type that is not already defined in the language itself.

    Python provides at least three different ways to import modules. You can use the import statement, the from statement, or the builtin __import__ function:

    1. import X
      imports the module X, and creates a reference to that module in the current namespace. Or in other words, after you've run this statement, you can use X.name to refer to things defined in module X.
    2. from X import *
      imports the module X, and creates references in the current namespace to all public objects defined by that module (that is, everything that doesn't have a name starting with "_"). Or in other words, after you've run this statement, you can simply use a plain name to refer to things defined in module X. But X itself is not defined, so X.name doesn't work. And if name was already defined, it is replaced by the new version. And if name in X is changed to point to some other object, your module won't notice.
    3. from X import a, b, c
      imports the module X, and creates references in the current namespace to the given objects. Or in other words, you can now use a and b and c in your program.
    4. X = __import__('X')
      works like import X, with the difference that you pass the module name as a string, and explicitly assign it to a variable in your current namespace.

    OOP classes and CSS classes; both of them represent general and specific "definitions", HTML elements can use multiple classes just as objects can inherit from multiple super classes, and the more poorly thought out either is, the harder it is to scale and extend the current code base.
    CSS classes also use abstraction(making things more general/simpler/abstract) and encapsulation(grouping together of information about an object) within HTML. A class creates a re-usable set of styles that can be applied anywhere in the HTML. Someone working in the html alone doesn't need to see all the details of the class. They just need to know that by adding class="frame" to any HTML element, they get a consistent frame style applied.
    These make the complex easier to work with and maintain, reduce the duplications and errors, make the code faster and re-usable, make easier to improve the code.

    Terminology

    Abstraction :

    The word "abstraction" in general refers to the process of dealing with things independently of their associations. In the world of programming, abstraction can come in many forms like data abstraction, code abstraction, control abstraction, etc.It reduces duplication of information in the code, makes the system more robust and tolerant to future changes.

    Attributes:

    Attributes are characteristics of an object. A special method called __init__() is used to initialize the attributes of an object.
    Pre-defined Class Attributes are "__dict__", "__name__", "__bases__", "__doc__", "__module__".
    Pre-defined Instance Attributes are "__dict__", "__class__"

    Class:

    Simply a logical grouping of data and functions ("methods" when defined within a class). A class can contain any data , and can have any functions (methods) attached to it. Rather than just throwing random things together under the name "class", classes are created where there is a logical connection between things.
    The class statement creates a new class definition. The name of the class immediately follows the keyword class followed by a colon.

    Class variable:

    A variable that is shared by all instances of a class. Class variables are defined within a class but outside any of the class's methods. Class variables are not used as frequently as instance variables are.

    Constructor:

    A constructor in a class is a special type of sub-routine called to create an object. It prepares the new object for use, often accepting arguments that the constructor uses to set required member variables.

    Encapsulation:

    The terms encapsulation and abstraction are often used as synonyms. They are nearly synonymous, i.e. abstraction is achieved though encapsulation. Generally speaking, encapsulation is the mechanism for restricting the access to some of an object's components, this means that the internal representation of an object can't be seen from outside of the objects definition. Access to this data is typically only achieved through special methods: Getters and Setters. By using solely get() and set() methods, we can make sure that the internal data cannot be accidentally set into an inconsistent or invalid state.

    Error handling:

    Python provides two very important features to handle any unexpected error in your Python programs and to add debugging capabilities in them:

    • Exception Handling (An exception is an event, which occurs during the execution of a program that disrupts the normal flow of the program's instructions. In general, when a Python script encounters a situation that it cannot cope with, it raises an exception. An exception is a Python object that represents an error.)
    • Assertions (An assertion is a sanity-check that you can turn on or turn off when you are done with your testing of the program. An expression is tested, and if the result comes up false, an exception is raised.).

    Inheritance :

    While Object Oriented Programming is useful as a modeling tool, it truly gains power when the concept of inheritance is introduced. Inherticance is the process by which a "child" class derives the data and behavior of a "parent" class.

    Instance:

    An individual object of a certain class. An object obj that belongs to a class Circle, for example, is an instance of the class Circle.

    Instance variable:

    A variable that is defined inside a method and belongs only to the current instance of a class.

    Instantiation:

    The creation of an instance of a class.

    Library:

    In programming, abstractions can make or break productivity. That's why, commonly used functions are collected into libraries which can be re-used by others. The programmers don't have to worry about the details, only need to understand how to use the ready-made abstractions. Obviously, this makes the code easier, faster and more productive.
    Python has a large standard library, commonly cited as one of Python's greatest strengths, providing tools suited to many tasks. This is deliberate and has been described as a "batteries included" Python philosophy. Some of these can send email, or fetch web pages. One of them is called "turtle" that is part of the standard python installation. Turtle graphics is a popular simplest way for introducing programming. Another example would be "os", provides a portable way of using operating system dependent functionality, such as os.name or os.path.

    Method:

    A special kind of function that is defined in a class definition. Methods (or message) are just like functions, with two differences:

    • Methods are defined inside a class definition in order to make the relationship between the class and the method explicit.
    • The syntax for invoking a method is different from the syntax for calling a function.

    Object:

    A unique instance of a data structure that's defined by its class. An object comprises both data members (class variables and instance variables) and methods.

    Operator overloading:

    The assignment of more than one function to a particular operator.

    Polymorphism:

    The polymorphism is the process of using an operator or function in different ways for different data input. In practical terms, polymorphism means that if class B inherits from class A, it doesn't have to inherit everything about class A; it can do some of the things that class A does differently.

    ***Networks & Servers & Forms & DataBases & MORE

    A Network is a group of entities (computers, servers, mainframes, network devices, peripherals, etc) that can communicate, even though they are not all directly connected.
    Typical scenarios that are especially sensitive to network speed issues include:
    • time to establish a new connection
    • time to load a Web Page
    • time to download an app, operating system patch, or other files
    • ability to stream video content for long periods without glitches
    Important Network Performance measures:

    Bandwidth:

    Amount of information that can be transmitted per unit time. It represents the overall capacity of the connection. Bandwidth is a key factor in determining the speed of a computer network. Virtually everyone knows the bandwidth ratings of their network routers and their Internet service. Commonly measured in bits/second.

    Latency:

    Time (delay) that takes message to get from source to destination, this is mainly a function of the signals travel time, and processing time at any nodes the information traverses.

    Throughput:

    Actual rate that information is transferred. It is controlled by available bandwidth, as well as the available signal-to-noise ratio and hardware limitations.

    Jitter:

    Variation in the time of arrival at the receiver of the information.

    Error Rate:

    Number of corrupted bits expressed as a percentage or fraction of the total sent.
    A bit is the basic unit of information in computing and digital communications. A bit can have only one of two values, these values are most commonly represented as either a 0 or 1. The two values can also be interpreted as logical values (true/false, yes/no), algebraic signs (+/−), activation states (on/off), or any other two-valued attribute.

    If Internet can be thought as serious of tubes, Latency is the length of the tube between 2 points, Bandwidth is how wide the tube is, the wider the tube, more data is sent, but still takes time to send, which is Latency. That is often confused.

    Operating a Network:
    • encoding and interpretting messages : messages -> bits -> electrons/protons
    • routing messages : a message comes in with a destination, routes have to figure out the next destination
    • ruling & deciding who gets to use resources
    A Protocol is the special set of rules that end points in a telecommunication connection use when they communicate. Protocols specify interactions between the communicating entities; client (web browser) and server (web server).The protocol, used on the web, is called HTTP(Hyper Text Transfer Protocol), which is very simple; only two main messages: Request & Response.
    URL (Uniform Resource Locator) is used to specify addresses on the World Wide Web, is the fundamental network identification for any resource connected to the web:
    protocol://domain:port/path?query_string#fragment_id
    example : http://example.com:80/toys?=foo#blah
    A Cache is a place to store something temporarily in a computing environment. The main purpose of a cache is to accelerate your computer while keeping the price of ­the computer low, it has some maximum size that is much smaller than the larger storage area. Caching allows you to do your computer tasks more rapidly.

    After client connects to the URL, it sends requests, asking the server to give responses:

    • Request : GET /foo HTTP/1.1 (METHOD /relativepath HTTP/version)
      Differences between GET and POST Methods :
      • GET include parameters in the URL and POST include the data in the request body.
      • GET is used for requesting, fetching documents, POST is used for sending data to the server.
      • GET parameters are generally okay to cache, POST parameters are almost never cached
      • GET requests should not change the Server, POST requests are okay to change the Server
    • Response : HTTP/1.1 200 OK (HTTP/version statuscode reasonphrase)
      Status codes : 100 Continue, 101 Switching Protocols, 2xx Successful, 3xx Redirection, 4xx Client Error, 5xx Server Error
    They both are followed by headers, such as Host, User Agent, Date, Content-Type, with a format of "Name: value".

    The purpose of a server is to respond to HTTP requests. There are 2 types of responses :
    1. Static : Static content is published to regular files on your server and handled using the simplest methods available to the web server. Fastest, most efficient, simpliest way, but it must be republished when it changes.
    2. Dynamic : Content is generated for you at the time you request the page. Good for pages change quickly, display viewer-specific. But it is resource-intensive, can display security problems, it cannot be used all pages. Most of the contents online are dynamic.
    A web application lives on a web server, speaks HTTP, and generates content that your browser requests.
    A HTML form on a web page allows a user to enter data that is sent to a server for processing. Forms can resemble paper or database forms because web users fill out the forms using checkboxes, radio buttons, or text fields.
    Validation basically means verifying on the Server side, so received data is what is expected to receive. This prevents the website from being hacked (gaining unauthorized access) and receiving junk data. All inputs should be treated as potentially dangerous, and validated to ensure that they contain the right information. A website asking user's birthday would be a good example, the server expects to see date format (month, day, year), if junk data is received unintentionally or intentionally, that can create a security hole(vulnerability) which allows unauthorized users (hackers) to gain access to a system or network.
    String substitution is used to avoid repetition. A function substitutes 1 variable into a string, so we don't have to build that string over and over again.
    Escaping in HTML is used to convert certain characters, such as HTML tags that can modify a website's appearance. Importing cgi in Python will apply the escape command where needed.
    Redirection is used to allow sharing success link and reloading page. It directs user to a success page, allows to have distinct pages for forms and successes. Reloading the success page doesn't resubmit the form.
    Using templates is a very efficient way of avoiding time consuming code repetition. A template library is a library to build complicated strings of HTML, such as jinja2, Django, Quik, Genshi. Templates make the code cleaner, more organized, more readable, easier to modify, secure, allow to seperate different type of codes, and to use inheritance & abstraction (maintaining one base html file across all webpages to avoid repetition).
    Variable Substitution in templates allows you to use variables to populate values in an HTML document using tags. Statement Syntax and Loop Syntax are methods for achieving this kind of substitution.
    Helpful tips:
    • always automatically escape variables when possible
    • minimize the code in templates
    • minimize html in code
    DataBase is a program that stores and returns large amounts of structured data, or it can refer to the machine running that program, or it can refer to a system of machines running this program operating together. Types of databases include Relational (postgreSQL, MySQL, SQLite, Oracle), Google App Engine Data Store (Dynamo, NoSQL) which uses objects instead of tables. Indexing help increase the speed of database reads, decreases the speed of database inserts and allow data to be sorted.
    Transactions are basically just a group of statements.
    ACID :
    • Atomicity : all parts of a transaction succeed or fail together
    • Consistency : the database will always be consistent
    • Isolation : no transaction can interfere with another
    • Durability : once the transaction is comitted, it won't be lost
